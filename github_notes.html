Бележки за github

Създаване на репозитория

  1. Логваш се в https://github.com/
  2. Отваряш падашото меню + в горния десен ъгъл на сайта
  3. Избираш опцията New repository
  4. В полето Repository name се пише името на репозиторията
  5. В полето Description се пише описание на репозиторията
  6. Трябва да има отметка в радио бутона Public
  7. Поставя се отметка в Initialize this repository with a README
  8. Натиска се зеления бутон Create repository

Клониране на репозитория

  1. Логваш се в github https://github.com/
  2. Цъкаш на Repositories
  3. Цъкаш върху името на репозиторията
  4. Натискаш зеления бутон Clone or download
  5. Цъкаш Use HTTPS (ако пише Use SSH пропускаш стъпката)
  6. Копираш генерирания линк https://github.com/u_name/name_of_repository.git
  7. Отваряш gitBash на твоя компютър
  8. Отиваш в папката, която ще съдържа работното копие на репозиторията
  9. Пишеш git clone и пействаш линка, който си копирал в стъпка 6

Това създава папка с името и съдържанието на репозиторията.
Изпълнява се само веднъж, когато започваш да работиш по проекта.
Може след като се пушнат всички промени да изтрие цялата папка, а когато е необходимо да се работи по проекта отново да се клонира от репозиторията

Изпращане на файлове в гит

  1. Отваряш gitBash
  2. Отиваш в папката на проекта
  3. Пишеш git status
  4. В червено са показани файловете, които са променени/добавени
  5. С git add ime_na_fajl.razsh добавяш файловете, които искаш да изпратиш
  6. С git commit -m"opisanie na komita" подготвяш пратката
  7. С git push изпращаш комита на github (ще иска усер наме и парола)
още

Все едно git add слага листове в плик за писмо
git commit -m"" надписва плика на писмото
git push - пъха писмото в пощенската кутия

Ако забравиш ключа -m"opis na kamita" gitBash ще се сбози натискаш Esc след това wq и ентер

Синхронизиране с репозиторията

  1. Отваряш gitBash
  2. Отиваш в папката на проекта
  3. Пишеш git pull
още

Тази процедура синхронизира работната папка с репозиторията т.е. от репозиторията се изтеглят файловете, които са били изменени след последния pull

Изтриване на репозитория

  1. Логваш се в github https://github.com/
  2. Цъкаш на Repositories
  3. Цъкаш върху името на репозиторията
  4. Цъкаш на Settings
  5. Отиваш най-отдолу в карето Danger Zone
  6. Натискаш бутона Delete this repository
  7. В полето за потвърждение пишеш името на репозиторията
  8. Натискаш бутона I understand the consequnces, delete this repository
  9. Натискаш зеления бутон Confirm password

Добавяне на съобщник

  1. Логваш се в github https://github.com/
  2. Цъкаш на Repositories
  3. Цъкаш върху името на репозиторията
  4. Цъкаш на Settings
  5. Цъкаш на Collaborators
  6. В полето Search пишеш името на съобщника
  7. Ако се намери натискаш бутона Add collaborator
  8. Съобщникът получава email с линк за потвърждение
  9. След 24 часа съобщникът ще може да потвърди присъединяването
още

Имало някакво 24 часово ограничение за добавяне на съобщник.
При первото добавяне действително колаборатора не можеше да потвърди 24 часа, но при следващите добавяния всичко ставаше веднага.
